# Lumen: Echoes of the Dusklands — Art Bible

The visual contract for the game. **Lumen** is a small luminous wisp who drifts
through the **Dusklands**: twilight stone ruins adrift over a dark void, lit by a
glow aesthetic that leans on opacity, sparkle, and soft warm/cool light.

Every asset is **original** and generated by `scripts/gen-art.mjs` — a pure-Node,
zero-dependency, deterministic PNG generator (re-run `npm run gen:art` and the
output is byte-identical). The PNGs live in `public/assets/lumen/` and are
**committed** (not synced from `@gitcade/library`). Vite serves `public/` at root,
so every file resolves at the URL **`assets/lumen/<name>.png`** — exactly the
`sprite.src` / `tilemap.tileset` strings used below.

This file is the contract the game-build session consumes: it fixes the palette,
the shape language for every element, the hero clip map, the tileset index→meaning
map, and the exact `shape` specs for elements drawn without a PNG.

---

## Palette

An original twilight set (9 colors). It is **not** the library's Sweetie16 subset
— a cool deep-indigo ground ramp, teal-lit stone, a warm amber hero, gold
collectibles, magenta danger, violet rifts. Use these consistently; the color
**coding** is itself gameplay-readable (warm = you/treasure, teal = safe footing,
magenta = danger, violet = arcane).

| Name       | Hex       | Role |
|------------|-----------|------|
| `void`     | `#070512` | the void / kill-plane, outlines, deepest shadow, riftgate core |
| `dusk`     | `#15103a` | primary background indigo/navy |
| `twilight` | `#2c2660` | drifting stone body — ground, platforms, ledges, decor |
| `aqua`     | `#34c2b3` | teal/cyan **lit** stone surfaces, ladders, rim light, driftstones |
| `ember`    | `#ff9a3c` | Lumen's warm amber core; emberstone core |
| `glow`     | `#ffe0a8` | pale warm light — Lumen's halo, eye glints, beacon light, highlights |
| `gold`     | `#ffc12e` | motes (coins), emberstone facets, HUD accents |
| `fuchsia`  | `#ff3fa4` | gloomspikes, driftwraiths — **danger** |
| `violet`   | `#9a5cff` | riftgates (portals), arcane "echoes" |

**Glow technique:** auras are soft (semi-transparent) radial pixels under an
opaque body; the host sets `image-rendering: pixelated`, so keep frames small and
crisp.

## The hero — Lumen (`player.png`)

**Silhouette:** a round glowing amber orb with a small **crown flame** on top and
two large, expressive dark eyes with a pale glint. A soft `ember` halo surrounds
it; a single cool `aqua` rim pixel on the lower-right ties it to the world's light.
Drawn **facing right** — the game's `face-velocity` behavior mirrors it for
leftward travel, so eyes and lean always point the way Lumen moves.

**Sheet:** `240×24`, frame size **24×24**, **10 frames**, single horizontal row
(row-major). Animation clips (inclusive frame ranges):

| Clip   | Frames | fps | loop  | Pose |
|--------|--------|-----|-------|------|
| `idle` | 0–1    | 4   | true  | settled → gentle bob, flame flicker |
| `run`  | 2–5    | 12  | true  | forward lean + bob, lengthening spark trail |
| `jump` | 6      | —   | false | stretched tall, eyes up, flame high |
| `fall` | 7      | —   | false | squashed wide, eyes wide-down, sparks trailing up |
| `land` | 8–9    | 18  | false | big squash + dust arc → recover |

Reference it in a scene as:

```json
{
  "kind": "sheet",
  "src": "assets/lumen/player.png",
  "frameWidth": 24,
  "frameHeight": 24,
  "frameCount": 10,
  "fps": 10,
  "animations": {
    "idle": { "from": 0, "to": 1, "fps": 4 },
    "run":  { "from": 2, "to": 5, "fps": 12 },
    "jump": { "from": 6, "to": 6, "loop": false },
    "fall": { "from": 7, "to": 7, "loop": false },
    "land": { "from": 8, "to": 9, "fps": 18, "loop": false }
  }
}
```

These clip names match the library's `sprite-state-machine` platformer states, so
attaching it drives the sheet for free.

## The world tileset (`tiles.png`)

**Style:** all of it is carved from one drifting twilight stone — `twilight` body,
`aqua` teal-lit surfaces (cap / walk-edge / rails), `void` shadow on the underside
and right, with seeded `aqua`/`glow` glimmers so a wall reads as glimmering dusk
stone rather than a flat slab.

**Sheet:** `192×32`, tile size **32×32**, **6 tiles**, single horizontal row. The
index→meaning map is a **frozen contract** — the scene's `tiles[]` indices mean:

| Idx | Meaning | Shape language | Intended collision role |
|-----|---------|----------------|-------------------------|
| 0 | **solid stone block** | opaque block, teal-lit cap + left edge, void bottom/right, brick seams | `solid` (full AABB) |
| 1 | **one-way ledge** (top-lit plank) | thin plank near the cell top, strong `aqua` lit edge, **transparent below** | `oneWay` (land from above, drop through) |
| 2 | **slope-left** | right-triangle, **high on the left**, lit diagonal descends to the right, transparent above | walkable ramp (see note) |
| 3 | **slope-right** | mirror of 2 — **high on the right**, descends to the left | walkable ramp (see note) |
| 4 | **ladder** | two `aqua` rails + pale `glow` rungs on a faint backing, mostly transparent | climbable (see note) |
| 5 | **decorative / background stone** | darker mottled `dusk`→`void` block, arcane `violet`/`aqua` flecks, **no lit cap** (reads as *behind*) | non-solid backdrop |

Reference it in a scene's tilemap:

```json
"tilemap": {
  "tileSize": 32,
  "tileset": "assets/lumen/tiles.png",
  "cols": 50, "rows": 15,
  "tiles": [ /* row-major indices; -1 = empty */ ],
  "properties": {
    "0": { "solid": true },
    "1": { "oneWay": true },
    "5": { }
  }
}
```

**Note on slopes & ladders:** indices 2–4 are provided as **visuals** plus the
conventional collision intent above. Whether the engine resolves slope ramps and
ladder-climbing is the game/engine session's call; `solid` (0) and `oneWay` (1)
are the well-trodden platformer tiles. Optional `properties[idx].color` (e.g.
`"0": { "color": "#2c2660" }`) only tints the headless/while-loading fallback —
the `tileset` image is the real render.

## Elements drawn as `shape` primitives (no PNG)

The game can render these with the engine's shape sprites — no asset needed. Use
these exact specs so they stay on-palette:

| Element | Lore name | Sprite spec | Notes |
|---------|-----------|-------------|-------|
| **spikes** | gloomspikes | `{ "kind": "shape", "shape": "triangle", "color": "#ff3fa4", "stroke": "#070512", "strokeWidth": 1 }` | one upward triangle = one spike (apex up); tile a row across a hazard span. `fuchsia` = the universal danger tell |
| **moving platform** | driftstones | `{ "kind": "shape", "shape": "rect", "color": "#2c2660", "stroke": "#34c2b3", "strokeWidth": 2 }` | a twilight slab with a teal-lit edge — matches the tileset stone; pair with a `path`/`tween` mover |
| **checkpoint** | — | inactive `{ "kind": "shape", "shape": "rect", "color": "#9a5cff" }` → on activation swap `color` to `"#34c2b3"` (or pulse opacity) | a slender standing marker; violet (dormant) → aqua (lit) reads as "saved" |
| **kill-plane** | the void | `{ "kind": "none" }` trigger entity, or a world-bounds death | invisible logic; the *visual* cue is the `void` `#070512` darkness opening below the lowest platforms |

## Background, void & HUD

- **Scene background:** base color `dusk` `#15103a`. Let the world open downward
  into `void` `#070512` below the lowest footing so falling reads as plunging into
  the void. (No background PNGs are shipped; a parallax layer set is optional
  future work — the engine supports `background.layers` if added.)
- **HUD / text:** text sprites bound to `world.state`. Counters in `gold`
  `#ffc12e` (motes/emberstones), primary labels in `glow` `#ffe0a8`, warnings in
  `fuchsia` `#ff3fa4`; monospace. Suggested fonts: `"16px monospace"` for HUD
  readouts, `"40px monospace"` for title/overlay headings. For a glow lift, draw a
  `void` `#070512` copy 1px down-right under the bright text.

```json
{ "kind": "text", "bind": "motes", "font": "16px monospace", "color": "#ffc12e", "align": "left" }
```
