# Lumen: Echoes of the Dusklands — Art Bible

The visual contract for the game. **Lumen** is a small luminous wisp who drifts
through the **Dusklands**: twilight stone ruins adrift over a vast dusk sky, lit by a
glow aesthetic that leans on opacity, sparkle, and soft warm/cool light.

**Art direction — "LUMINOUS DUSK."** The world is keyed VIVID, LAYERED, and
HIGH-CONTRAST, the way modern 2D platformers (Celeste / Ori / Rayman Legends) read —
not flat-dark. The lever is the modern-2D depth trick: the **PLAY layer** (tiles,
platforms, props) is saturated, lifted off near-black, and POPS, while the **BACKGROUND
layers** fade back — desaturated, cooler, lower-contrast (atmospheric perspective). The
light-vs-dark "Echo" identity and the color coding are unchanged; the value range is
just opened up so the foreground sings.

Every asset is **original** and generated by `scripts/gen-art.mjs` — a pure-Node,
zero-dependency, deterministic PNG generator (re-run `npm run gen:art` and the
output is byte-identical). The PNGs live in `public/assets/lumen/` and are
**committed** (not synced from `@gitcade/library`). Vite serves `public/` at root,
so every file resolves at the URL **`assets/lumen/<name>.png`** — exactly the
`sprite.src` / `tilemap.tileset` / `background.layers[].src` strings used below.

This file is the contract the game-build session consumes: it fixes the palette,
the shape language for every element, the hero clip map, the tileset index→meaning
map, the parallax background-layer set, and the exact `shape` specs for elements
drawn without a PNG.

## Palette

An original luminous-dusk set, **not** the library's Sweetie16 subset. The color
**coding** is gameplay-readable and unchanged: **warm = you/treasure, teal = safe
footing/light, magenta·fuchsia = danger, violet = arcane.** The set is grouped by the
depth role it plays (`gen-art.mjs`'s `HEX` is the single source of truth):

**Hero-locked anchors** — these five feed `drawLumen()`; the hero is **frozen** this
pass, so they are byte-load-bearing (changing one drifts `player.png`):

| Name    | Hex       | Role |
|---------|-----------|------|
| `void`  | `#070512` | outlines, deepest shadow, kill-plane, riftgate core |
| `aqua`  | `#34c2b3` | hero's cool rim pixel + landing dust (world lit caps use `litcap`) |
| `ember` | `#ff9a3c` | warm amber — hero core; emberstone / beacon warmth |
| `glow`  | `#ffe0a8` | pale warm light — halos, glints, beacon, highlights |
| `gold`  | `#ffc12e` | hero's gold inner + crown flame (world collectibles use `coin`) |

**Play layer** — saturated, bright, high-contrast so it POPS in front:

| Name       | Hex       | Role |
|------------|-----------|------|
| `twilight` | `#3a2f6b` | drifting play-stone body — lifted off near-black |
| `litcap`   | `#4fe0cf` | bright aqua **lit** cap / walk-edge / rails — the "this is solid" tell |
| `rim`      | `#9af0e4` | pale teal rim sheen (1px lip above the lit cap) |
| `coin`     | `#ffd24a` | bright coding gold — motes, emberstone facets, HUD accents |
| `fuchsia`  | `#ff4fb0` | gloomspikes, driftwraiths — **danger** |
| `violet`   | `#9a5cff` | riftgates (portals), arcane "echoes" |

**Sky + atmosphere** — the background layers; desaturated/cooler, they fade back:

| Name         | Hex       | Role |
|--------------|-----------|------|
| `dusk`       | `#1b1442` | scene base color + sky **top** (deep indigo) |
| `skyPurple`  | `#5b2a8a` | sky mid band |
| `skyMagenta` | `#b5417a` | sky dusk band |
| `horizon`    | `#ff9a4d` | warm amber horizon glow |
| `haze`       | `#6a5a9c` | cool atmospheric haze the distant layers fade toward |

**Glow technique:** auras are soft (semi-transparent) radial pixels under an
opaque body; the host sets `image-rendering: pixelated`, so keep frames small and
crisp.

**Depth technique (atmospheric perspective):** the farther a background layer sits,
the more its colors are pulled toward `haze`/the sky and the lower its internal
contrast — and crucially, **no background layer uses the `void` shadow or the bright
`litcap`/`rim` edge.** That full-range pairing (near-black underside ↔ bright cyan cap)
belongs to the play layer alone, which is what makes it read as *in front*.

## The hero — Lumen (`player.png`)

> **Frozen this pass.** The luminous-dusk re-key is "world first": `player.png` is
> kept **byte-identical** (its draw routine and the five hero-locked anchors are
> untouched). The warm amber hero already pops against the brighter world, so it was
> deliberately left alone — a hero re-key, if ever wanted, is a separate deliberate pass.

**Silhouette:** a round glowing amber orb with a small **crown flame** on top and
two large, expressive dark eyes with a pale glint. A soft `ember` halo surrounds
it; a single cool `aqua` rim pixel on the lower-right ties it to the world's light.
Drawn **facing right** — the game's `face-velocity` behavior mirrors it for
leftward travel, so eyes and lean always point the way Lumen moves.

**Sheet:** `240×24`, frame size **24×24**, **10 frames**, single horizontal row
(row-major). Animation clips (inclusive frame ranges):

| Clip   | Frames | fps | loop  | Pose |
|--------|--------|-----|-------|------|
| `idle` | 0–1    | 4   | true  | settled → gentle bob, flame flicker |
| `run`  | 2–5    | 12  | true  | forward lean + bob, lengthening spark trail |
| `jump` | 6      | —   | false | stretched tall, eyes up, flame high |
| `fall` | 7      | —   | false | squashed wide, eyes wide-down, sparks trailing up |
| `land` | 8–9    | 18  | false | big squash + dust arc → recover |

Reference it in a scene as:

```json
{
  "kind": "sheet",
  "src": "assets/lumen/player.png",
  "frameWidth": 24,
  "frameHeight": 24,
  "frameCount": 10,
  "fps": 10,
  "animations": {
    "idle": { "from": 0, "to": 1, "fps": 4 },
    "run":  { "from": 2, "to": 5, "fps": 12 },
    "jump": { "from": 6, "to": 6, "loop": false },
    "fall": { "from": 7, "to": 7, "loop": false },
    "land": { "from": 8, "to": 9, "fps": 18, "loop": false }
  }
}
```

These clip names match the library's `sprite-state-machine` platformer states, so
attaching it drives the sheet for free.

## The world tileset (`tiles.png`)

**Style:** all of it is carved from one drifting twilight stone — a brighter
`twilight` body with a **chunky, high-contrast** triple-row lit cap (a `rim` lip → a
`litcap` band → a 1px falloff) on the cap / walk-edge / rails, a hard `void` shadow on
the underside and right, and seeded `litcap`/`glow` glimmers. That strong cap↔shadow
range is what makes a block read as unmistakably SOLID and POP off the dusk sky. Only
the decor stone (5) is held back — dim, cool, capless — so it reads as *behind*.

**Sheet:** `192×32`, tile size **32×32**, **6 tiles**, single horizontal row. The
index→meaning map is a **frozen contract** — the scene's `tiles[]` indices mean:

| Idx | Meaning | Shape language | Intended collision role |
|-----|---------|----------------|-------------------------|
| 0 | **solid stone block** | opaque block, bright `litcap`/`rim` cap + lit left edge, hard `void` bottom/right, chunky brick seams | `solid` (full AABB) |
| 1 | **one-way ledge** (top-lit plank) | thin plank near the cell top, strong `litcap` lit edge + `rim` lip, **transparent below** | `oneWay` (land from above, drop through) |
| 2 | **slope-left** | right-triangle, **high on the left**, bright lit diagonal descends to the right, transparent above | walkable ramp (see note) |
| 3 | **slope-right** | mirror of 2 — **high on the right**, descends to the left | walkable ramp (see note) |
| 4 | **ladder** | two `litcap` rails (`rim` highlight) + pale `glow`/`coin` rungs on a faint backing, mostly transparent | climbable (see note) |
| 5 | **decorative / background stone** | dim, cool, desaturated `dusk`→`void` block, arcane `violet`/`litcap` flecks, **no lit cap** (reads as *behind*) | non-solid backdrop |

Reference it in a scene's tilemap:

```json
"tilemap": {
  "tileSize": 32,
  "tileset": "assets/lumen/tiles.png",
  "cols": 50, "rows": 15,
  "tiles": [ /* row-major indices; -1 = empty */ ],
  "properties": {
    "0": { "solid": true },
    "1": { "oneWay": true },
    "5": { }
  }
}
```

**Note on slopes & ladders:** indices 2–4 are provided as **visuals** plus the
conventional collision intent above. Whether the engine resolves slope ramps and
ladder-climbing is the game/engine session's call; `solid` (0) and `oneWay` (1)
are the well-trodden platformer tiles. Optional `properties[idx].color` (e.g.
`"0": { "color": "#3a2f6b" }`) only tints the headless/while-loading fallback —
the `tileset` image is the real render.

## Parallax background layers (`sky.png` · `far.png` · `mid.png` · `near.png`)

Four background planes give the Dusklands real depth. All are **256 px wide** and
**480 px tall** (the intended viewport height), authored to **tile seamlessly
left↔right** and to fill a 480-tall viewport with **one** vertical copy — content is
**bottom-anchored** with transparency above (the sky is the only opaque plane). The
farther the layer, the more it fades toward the sky / `haze` (desaturated, cooler,
lower contrast), so the play layer pops in front.

| File | Size | Opacity | Content | Intended parallax factor |
|------|------|---------|---------|--------------------------|
| `sky.png`  | 256×480 | **opaque** | the dusk gradient (`dusk` → `skyPurple` → `skyMagenta` → `horizon`) + faint stars | `0` (static) |
| `far.png`  | 256×480 | transparent above | distant rolling peaks + drifting spires, **most faded** | `~0.2` |
| `mid.png`  | 256×480 | transparent above | mid drifting-ruin towers with faint lit windows | `~0.5` |
| `near.png` | 256×480 | mostly transparent | near mist + soft drifting light-wisps | `~0.8` |

**How the engine drives them.** The built-in `background.layers` slot
(`{ color, layers: [{ src, scrollX, scrollY }] }` on a scene's `background`) tiles each
plane across the viewport and drifts it by `scrollX`/`scrollY` **× `world.time`** — a
constant time-based drift, **NOT** coupled to camera position. So:

- Use the built-in slot for an ambient time-drift (give back→front layers increasing
  `scrollX`) and you get a living dusk sky for free, no host glue.
- For **true camera-coupled parallax** (layers shifting with where the player *is*), the
  game-wiring session drives the offsets itself from the camera. These PNGs are built to
  be used **either way**; the factors above are the relative speeds to aim for.

Reference the built-in slot in a scene (layers paint back→front in array order, over the
`color` fill; `color` `#1b1442` matches `sky.png`'s top so any gap reads as more sky):

```json
"background": {
  "color": "#1b1442",
  "layers": [
    { "src": "assets/lumen/sky.png",  "scrollX": 0 },
    { "src": "assets/lumen/far.png",  "scrollX": 6 },
    { "src": "assets/lumen/mid.png",  "scrollX": 15 },
    { "src": "assets/lumen/near.png", "scrollX": 24 }
  ]
}
```

## Collectibles & props (PNG sprites)

All props are re-keyed onto the luminous-dusk palette for cohesion — brighter `coin`
collectibles, brighter `fuchsia` danger, and stone bases/pedestals that match the
re-keyed tileset (brighter `twilight` body + bright `litcap` cap). Silhouettes and the
color coding are unchanged.

| Element (lore name) | File | Sprite kind | Silhouette |
|---------------------|------|-------------|------------|
| **mote** (coin) | `mote.png` `64×16` | `sheet` 16×16 ×4 | a glowing **gold** orb, 4-frame spin (width narrows to an edge sliver and back), warm rim + pale top glint + soft halo |
| **emberstone** (gem) | `emberstone.png` `16×16` | `image` | a **faceted warm crystal** — `gold`/`ember` diamond, glow upper facets, a hot near-white core, soft `ember` halo. Sharper & rarer-looking than a mote |
| **driftwraith** (patrolling enemy) | `driftwraith.png` `48×24` | `sheet` 24×24 ×2 | a **fuchsia** spectre: rounded top, **tattered tail** of three swaying lobes, menacing `void` eye-slits under a `violet` brow. 2-frame bob/sway |
| **riftgate** (portal) | `riftgate.png` `32×48` | `image` | a tall **violet** rift — concentric violet/glow rings around a `void` core, swirling sparks, anchored on a teal-lit stone base |
| **Beacon** (level goal) | `beacon.png` `32×64` | `image` | a **warm light column** (glow→ember) rising from a teal-lit stone pedestal, bright core, crown spark, rising gold sparks. Warm, distinct from the cool riftgate |

Reference patterns:

```json
{ "kind": "sheet", "src": "assets/lumen/mote.png", "frameWidth": 16, "frameHeight": 16,

{ "kind": "image", "src": "assets/lumen/emberstone.png" }   // entity size 16×16

{ "kind": "sheet", "src": "assets/lumen/driftwraith.png", "frameWidth": 24, "frameHeight": 24,
  "frameCount": 2, "fps": 3, "animations": { "bob": { "from": 0, "to": 1, "fps": 3 } } }

{ "kind": "image", "src": "assets/lumen/riftgate.png" }      // entity size 32×48
{ "kind": "image", "src": "assets/lumen/beacon.png" }        // entity size 32×64
```

## Elements drawn as `shape` primitives (no PNG)

The game can render these with the engine's shape sprites — no asset needed. Use
these exact specs so they stay on-palette:

| Element | Lore name | Sprite spec | Notes |
|---------|-----------|-------------|-------|
| **spikes** | gloomspikes | `{ "kind": "shape", "shape": "triangle", "color": "#ff4fb0", "stroke": "#070512", "strokeWidth": 1 }` | one upward triangle = one spike (apex up); tile a row across a hazard span. `fuchsia` = the universal danger tell |
| **moving platform** | driftstones | `{ "kind": "shape", "shape": "rect", "color": "#3a2f6b", "stroke": "#4fe0cf", "strokeWidth": 2 }` | a twilight slab with a bright lit edge — matches the re-keyed tileset stone; pair with a `path`/`tween` mover |
| **checkpoint** | — | inactive `{ "kind": "shape", "shape": "rect", "color": "#9a5cff" }` → on activation swap `color` to `"#4fe0cf"` (or pulse opacity) | a slender standing marker; violet (dormant) → lit-teal (active) reads as "saved" |
| **kill-plane** | the void | `{ "kind": "none" }` trigger entity, or a world-bounds death | invisible logic; the *visual* cue is the `void` `#070512` darkness opening below the lowest platforms |

## Background, void & HUD

- **Scene background:** base color `dusk` `#1b1442` (it matches `sky.png`'s top). The
  shipped **parallax layer set** (`sky`/`far`/`mid`/`near`, see above) is the real
  backdrop; let the world still open downward into `void` `#070512` below the lowest
  footing so falling reads as plunging into the void.
- **HUD / text:** text sprites bound to `world.state`. Counters in `coin`
  `#ffd24a` (motes/emberstones), primary labels in `glow` `#ffe0a8`, warnings in
  `fuchsia` `#ff4fb0`; monospace. Suggested fonts: `"16px monospace"` for HUD
  readouts, `"40px monospace"` for title/overlay headings. For a glow lift, draw a
  `void` `#070512` copy 1px down-right under the bright text.

```json
{ "kind": "text", "bind": "motes", "font": "16px monospace", "color": "#ffd24a", "align": "left" }
```
