# Arena Re-Skin — Phase 2B presentational proof

This is the Phase 2A `survival-arena mobs` demo **re-skinned** with the Phase 2B
presentational half. The *logic* is unchanged — the same four reuse parts
(`ai-chase` + `contact-damage` + `wave-spawner` + `health-and-death`) drive scaling
waves of mobs that the player clears with contact-damage thorns. What's new is how
it **looks and sounds**:

| Layer | Phase 2A (logic proof) | Phase 2B (this re-skin) |
|---|---|---|
| Entities | placeholder rectangles | generated sprites: `player-blob` (animated), `enemy-chaser`, spinning `coin` |
| World | flat color | `background-starfield` backdrop + `tileset-space` accent studs |
| Audio | SDK beeps | synthesized `LibraryAudioPlayer` — `explode`/`lose`/`collect` SFX + the `action` chiptune loop |
| FX | none | `explosion` particles on `enemy-died`, `sparkle` on `collect`, host-side screen-shake/flash |
| UI | none | `hud-score` (kills), `hud-wave-counter`, `hud-health-bar` |

Everything visual is procedurally generated by `packages/library/scripts/gen-assets.ts`
(deterministic PNGs on the fixed 8-color palette); all audio is synthesized at
runtime (zero binary audio assets).

## Run it

```bash
npm run dev        # vite dev server (auto-syncs assets into public/); open the URL
npm test           # headless smoke test (logic + fx integration)
npm run validate   # gitcade validate . (defers its smoke boot to npm test)
```

`public/assets/` is a build-time copy of the library's generated assets (kept out
of git; recreated by the `sync-assets` script the `dev`/`build` steps run first).
